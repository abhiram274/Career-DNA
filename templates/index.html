Complete Technical Documentation

NNDLConceptsUsed

Every neural network concept in CareerDNA — explained with what it does, where it is used, and the real code behind it.

🧠
Multi-Class ANN
→ train_model.py · build_model() · /api/career
A 4-layer feedforward ANN classifies a 30-dimensional skill vector into one of 14 career paths. The output layer has 14 neurons — one per career — and Softmax converts them to probabilities. This is the same architecture as ImageNet image classifiers.
Input(30) → Dense(128,ReLU) + BN + Drop(0.3) → Dense(64,ReLU) + BN + Drop(0.2) → Dense(32,ReLU) → Dense(14, Softmax) # 15,534 trainable params
📐
Cosine Similarity
→ app.py · cosine_similarity() · all routes
Measures the angle between two skill sets used as vectors. Used alongside ANN probability for career matching, and as primary engine for Interview IQ. cos(θ)=1 means perfect match. Same math as Word2Vec and BERT semantic search.
cos(θ) = (A·B) / (|A| × |B|) = |intersection| / (√|A| × √|B|) # 0 = no match, 1 = perfect match def cosine_similarity(a, b): inter = len(set(a) & set(b)) mag = len(a)**0.5 * len(b)**0.5 return inter / mag
⚡
ReLU Activation
→ train_model.py · all hidden layers
ReLU f(x)=max(0,x) introduces non-linearity. Without it, stacking Dense layers collapses to one linear equation — useless for complex patterns. ReLU also prevents vanishing gradients, keeping gradients at exactly 1 for positive inputs so deep networks can train.
def relu(x): return max(0, x) # Keras: layers.Activation('relu') # 3 hidden layers use ReLU # Output layer uses Softmax instead # Used in JS too: relu(score * weight)
🎲
Softmax Output
→ train_model.py output · app.py career route
Converts raw ANN logits to a probability distribution summing to 1. The career with the highest softmax output = predicted career. The match % shown is the softmax confidence score. Used identically in all multi-class classifiers.
softmax(xi) = exp(xi) / Σ exp(xj) # All outputs ∈ [0,1], sum = 1.0 # In app.py: def softmax(scores): exps = [exp(s * 3) for s in scores] total = sum(exps) return [e / total for e in exps]
🛡️
Dropout Regularisation
→ train_model.py · Dropout(0.3/0.2)
Randomly zeros 30%/20% of neurons per training batch. Forces the network to learn redundant distributed representations — cannot memorise training patterns. Without dropout, the ANN overfits to our synthetic dataset and gives wrong predictions on real user skills.
layers.Dropout(0.30) # 30% zeroed layer 1 layers.Dropout(0.20) # 20% zeroed layer 2 # Inverted: remaining * 1/(1-p) # OFF during model.predict() inference
📊
Batch Normalisation
→ train_model.py · BatchNormalization() × 2
Normalises layer outputs to mean≈0, std≈1 within each mini-batch. Stabilises training and allows Adam to use lr=0.001 safely. Without BN, our ANN would need careful lr tuning and would train 3–5× slower on the career classification task.
x = layers.Dense(128)(inp) x = layers.BatchNormalization() # μ≈0, σ²≈1 x = layers.Activation('relu') x = layers.Dropout(0.3) # γ, β are learnable per-feature params
📋
TF-IDF (Interview IQ)
→ app.py · tfidf_weight() · /api/interview
Weights each JD keyword by TF (how often it appears in this JD) × IDF (how rare it is across all JDs). This surfaces what the job truly requires vs. generic filler. A skill mentioned 4× in a niche JD scores higher than a common skill mentioned once.
TF(w,d) = count(w in doc) / total_words IDF(w) = log(N / docs_with_w) + 1 TF-IDF = TF × IDF # Converts raw JD text → weighted vector # Then cosine_sim(jd_vec, resume_vec) # = readiness score per topic
🎯
Adam Optimizer
→ train_model.py · Adam(lr=0.001)
Adam = Momentum + RMSProp. Each weight gets its own adaptive learning rate. Momentum stores gradient direction, RMSProp scales by recent gradient magnitude. Converges 5–10× faster than vanilla SGD on the career classification task.
m = β₁·m + (1-β₁)·g # momentum v = β₂·v + (1-β₂)·g² # RMSProp w = w - lr·m̂/(√v̂ + ε) # update # β₁=0.9, β₂=0.999, lr=0.001, ε=1e-7
📉
Cross-Entropy Loss
→ train_model.py · compile(loss=...)
Sparse Categorical Cross-Entropy is the loss for multi-class classification. Measures how wrong the ANN's probability distribution is vs. the true career label. Minimised via backpropagation over 150 epochs. L = −log(predicted_prob_of_true_class).
model.compile( loss='sparse_categorical_crossentropy', optimizer='adam', metrics=['accuracy'] ) # 'sparse' = integer labels, not one-hot # Perfect prediction: L → 0 # Wrong prediction: L → ∞
🔢
MinMax Normalisation
→ train_model.py · MinMaxScaler + app.py
Scales all features to [0,1] before ANN input. Without this, skill count 25 dominates velocity score 0.8 in gradient descent. Critical rule: fit scaler ONLY on training data. Applying it to test data before fitting = data leakage = inflated accuracy.
scaler = MinMaxScaler() X_train = scaler.fit_transform(X_train) X_test = scaler.transform(X_test) # fit_transform: learns min/max from TRAIN # transform: uses SAME params on TEST # NEVER fit on test = data leakage
